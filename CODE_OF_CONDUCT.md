# Code Of Conduct

## Our Standard

We want Panopticon to be a professional, technically serious, and respectful
project. Contributors should engage in ways that make the repo easier to reason
about and safer for others to participate in.

Expected behavior:

- discuss ideas directly and respectfully
- critique code and claims, not people
- keep disagreements technical and evidence-based
- respect maintainers' need to keep scope and safety boundaries clear

Unacceptable behavior:

- harassment, abuse, or intimidation
- bad-faith argumentation or deliberate misrepresentation
- doxxing or disclosure of private information
- hostile conduct that makes collaboration unsafe or unproductive

## Enforcement

Project maintainers may remove comments, issues, pull requests, or
participation that violate this code of conduct.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers
through the same private path used for security reporting.
